🚀 Vinted Scraper Bot

Starte in 5 Minuten - Alles was du brauchst steht hier

⚡ Super Quick Start (5 Min)

  1. Docker installieren

    Download: docker.com

    docker --version
  2. .env Datei erstellen

    Neue Datei mit Namen .env (NICHT .env.txt!)

    TELEGRAM_TOKEN=dein_token TELEGRAM_CHAT_ID=deine_id DISCORD_TOKEN=dein_discord_token DISCORD_CHANNEL_ID=dein_channel_id SUCHBEGRIFF=sneaker
    ⚠️ Noch keine Tokens? → Gehe zu TAB "TOKENS"
  3. Projekt entpacken & öffnen

    TAR entpacken (oder ZIP wenn vorhanden):

    tar -xf version_1_0_vinted.tar cd VintedSCRAPER
  4. Docker bauen
    docker build -t vinted-scraper .

    ⏱️ Dauert 1-2 Minuten

  5. Bot starten
    docker-compose up -d
    ✅ FERTIG! Bot läuft jetzt im Hintergrund!
  6. Logs prüfen
    docker-compose logs -f

    Sollte zeigen:

    ✅ Config geladen ✅ Discord Bot eingeloggt ✅ Telegram Token: True

🔑 Tokens holen

1️⃣ Telegram Token & Chat ID

  1. Öffne Telegram
  2. Suche: @BotFather
  3. Schreib: /newbot
  4. Gib einen Namen ein (z.B. "Mein Vinted Bot")
  5. BotFather sendet dir einen TOKEN
    123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11
  6. Kopiere diesen Token in .env als TELEGRAM_TOKEN=...
  7. Für TELEGRAM_CHAT_ID: Schreib @userinfobot und kopiere deine ID

2️⃣ Discord Token & Channel ID

  1. Gehe zu: Discord Developer Portal
  2. Klick: "New Application"
  3. Gib Namen ein (z.B. "Vinted Bot")
  4. Gehe zu "Bot" Tab
  5. Klick "Add Bot"
  6. Klick "Reset Token" → Kopieren → in .env als DISCORD_TOKEN=...
  7. Für DISCORD_CHANNEL_ID:
    • Öffne Discord
    • Rechtsklick auf Channel
    • "Copy Channel ID" → in .env einfügen
✅ Alle Tokens in .env einfügen?
→ Weiter zu TAB "DOCKER"

🐳 Docker Commands

Befehl Was es macht
docker build -t vinted-scraper . Image bauen (1x am Anfang)
docker-compose up -d Bot im Hintergrund starten
docker-compose down Bot stoppen & löschen

docker-compose restart Bot neu starten

🆘 Häufige Fehler

❌ "docker: command not found"

Docker nicht installiert. Download: docker.com

Nach Installation: Computer neustarten!

❌ "TELEGRAM_TOKEN nicht gefunden"

Deine .env Datei ist leer oder existiert nicht.

Prüfen:

  • .env existiert im Projekt-Ordner (neben Dockerfile)
  • .env ist NICHT .env.txt
  • .env hat 5 Zeilen (TOKEN, CHAT_ID, etc.)

Lösung: Neu bauen:

docker-compose down docker image prune -a docker-compose up -d --build
❌ "Container läuft nicht" / "Error"

Schau dir die Logs an:

docker-compose logs

Die Logs zeigen GENAU was falsch ist. Meist:

  • .env nicht gefunden
  • Tokens sind ungültig
  • Python Fehler im Code
❌ "Keine neuen Artikel gefunden"

Das ist NORMAL! Passiert wenn:

  • Keine neuen Artikel für deinen Suchbegriff vorhanden sind
  • Bot hat bereits alle als "gesehen" markiert
  • Warte einfach bis neue Artikel online gehen

Test: Ändere SUCHBEGRIFF in .env und starte neu

❌ "Cannot connect to Vinted"

Das ist NORMAL! Der Bot hat Retry-Logic:

  • Versucht 5x automatisch
  • Wartet nach jedem Fehler länger (5s, 10s, 20s, 40s, 80s)
  • Läuft danach einfach weiter

❌ KEIN Manual-Neustart nötig!
